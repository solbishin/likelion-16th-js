이벤트 전파 (Event Propagation)

이벤트가 발생했을 때 캡처링, 타겟, 버블링의 세 단계를 거쳐 DOM 트리를 여행하는 원리를 이해합니다.

이전 전파의 3단계

이벤트 전파 단계별 특징 요약
단계 (Phase) 방향 설명
1. 캡처링 (Capturing) 상위 → 하위 Window에서 타겟 요소까지 이벤트가 내려가는 단계
2. 타겟 (Target) 대상 도달 이벤트가 실제 발생한 요소에 도달하여 리스너 실행
3. 버블링 (Bubbling) 하위 → 상위 타겟에서 다시 Window까지 이벤트가 올라가는 단계

전파 제어 메서드

  • stopPropagation(): 상위 요소로의 전파(버블링)를 중단합니다.
  • stopImmediatePropagation(): 상위 전파 중단은 물론, 현재 요소에 등록된 다른 리스너 실행까지 모두 멈춥니다.
💡 리스너 주요 옵션
  • capture: true - 캡처링 단계에서 이벤트를 감지합니다.
  • once: true - 이벤트를 단 한 번만 실행하고 제거합니다.
  • passive: true - 스크롤 성능 최적화를 위해 브라우저의 기본 동작 차단을 포기합니다.

요약

이벤트 전파 원리를 알면 이벤트 위임(Delegation)과 같은 효율적인 기법을 사용할 수 있으며, 의도치 않은 상위 요소의 이벤트 실행을 방지할 수 있습니다.
