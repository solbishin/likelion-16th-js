이벤트 핸들러 내 요소 참조

이벤트 리스너 콜백 함수 내에서 이벤트를 수신한 요소를 변수 이름에 의존하지 않고 안전하게 참조하는 방법을 배웁니다.

요소 참조 방법 비교

이벤트 참조 방식인 this와 e.currentTarget 비교
구분 설명 주의사항
this 일반 함수에서 이벤트를 수신한 요소를 가리킴 화살표 함수에서는 사용 불가 (Window 객체 참조)
e.currentTarget 이벤트 리스너가 등록된 요소를 명확히 참조 함수 형태와 관계없이 항상 안전하게 사용 가능

화살표 함수 사용 시 주의사항

화살표 함수는 자신만의 this를 가지지 않고 상위 스코프의 this를 계승합니다.

⚠️ 경고

화살표 함수 내부에서 이벤트를 수신한 요소에 접근해야 한다면, this 대신 반드시 e.currentTarget을 사용하세요.

요약

변수 이름을 변경하더라도 코드가 망가지지 않도록 유연한 참조 방식을 선택하는 것이 유지보수가 쉬운 코드를 만드는 비결입니다.
