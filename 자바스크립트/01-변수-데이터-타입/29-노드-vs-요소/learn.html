호텔의 구조물과 방: 노드(Node)와 요소(Element)

DOM을 처음 접하면 노드와 요소라는 용어가 헷갈릴 수 있습니다. 호텔 전체를 지탱하는 뼈대(노드)와 손님이 머무는 실제 객실(요소)의 차이를 알아봅시다.

1. 노드(Node)란 무엇인가?

DOM을 트리 다이어그램(Tree Diagram)으로 시각화했을 때, 각 줄기와 잎(Leaf) 하나하나를 노드(Node)라고 부릅니다. DOM은 이 노드들이 모여 이룬 전체 지도와 같습니다.

위 이미지와 같은 트리 구조에서 우리는 다양한 노드를 발견할 수 있습니다.

  • <html>: 루트 요소 노드 (Root Element Node)
  • <body>, <h1>, <a>: 요소 노드 (Element Node)
  • "Link text": 텍스트 노드 (Text Node)

2. 노드의 종류

노드는 단순히 태그만을 의미하지 않습니다. 다음 코드를 실행하면 시스템에서 정의된 다양한 노드 유형을 확인할 수 있습니다.

for (const key in document) {
  if (key.includes('NODE')) console.log(key)
}

현재 웹 표준에서 주로 다루는 노드 목록은 다음과 같습니다. (일부는 더 이상 사용되지 않습니다.)

  • Element node (요소 노드)
  • Text node (텍스트 노드)
  • Document node / Document type node
  • Comment node (주석 노드)
  • Attribute node / CDATA section node (Deprecated)

3. 노드의 주요 속성과 메서드

모든 노드는 공통의 도구를 가지고 있습니다. 그중 실무에서 가장 자주 쓰이는 핵심 도구 두 가지를 기억하세요.

Node.parentElement
현재 노드를 감싸고 있는 부모 요소를 찾습니다. 없으면 null을 반환합니다.
Node.textContent
해당 노드와 그 자식 노드들에 포함된 모든 텍스트 내용을 가져오거나 수정합니다.

4. 요소(Element): 특별한 노드

요소(Element)는 노드의 한 종류이지만, 매우 특별합니다. 노드가 가진 모든 능력을 물려받으면서도, HTML 태그로서 가질 수 있는 classList, getAttribute와 같은 전용 무기를 추가로 가지고 있습니다.

지배인 메모: "모든 요소는 노드이지만, 모든 노드가 요소는 아닙니다." 주석이나 단순한 텍스트는 노드일 뿐, '요소'라고 부르지 않습니다.

5. 요약 및 비교

노드와 요소의 차이점 요약
항목 노드 (Node) 요소 (Element)
정의 DOM 트리를 구성하는 모든 단위 HTML 태그로 표현되는 특정 노드
포함 범위 텍스트, 주석, 문서 등 전체 포함 노드의 하위 개념 (특수한 노드)
주요 메서드 parentElement, textContent classList, querySelector
사용 빈도 기본 구조 이해용 (비교적 적음) 실제 DOM 조작의 대부분 (매우 높음)
