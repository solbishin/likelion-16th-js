호텔 자동화 시스템: 반복문(Loop) 마스터

수백 명의 손님에게 일일이 인사하기 힘드신가요? 루프(Loop)를 사용하면 반복되는 업무를 단 몇 줄의 코드로 자동화할 수 있습니다.

1. 루프(Loop)란 무엇인가?

명령어를 프로그램 내에서 일정하게 반복해서 실행하는 것을 말합니다. 조건이 충족될 때까지 멈추지 않고 일합니다.

지배인 주의: 조건이 항상 참(true)이면 호텔 시스템이 멈추지 않는 '무한 루프'에 빠지게 되니 주의하세요!

2. 상황별 반복문 선택 가이드

반복문의 유형 및 특징
유형 특징 적합한 상황
while 조건 평가 후 실행 반복 횟수가 불명확할 때
do ... while 실행 후 조건 평가 최소 1회 실행이 보장되어야 할 때
for 선언, 조건, 증감을 한곳에 정해진 횟수나 배열 순회 시 (가장 대중적)

3. 업무 흐름 제어하기

반복 업무 도중 예외 상황이 발생하면 다음 도구를 사용합니다.

  • continue: 현재 손님은 건너뛰고 바로 다음 손님으로 넘어가기
  • break: 즉시 업무를 중단하고 루프 밖으로 나가기

4. 장부(배열/객체) 전용 반복문

for ... of (배열 전용)
방 번호(인덱스)보다 손님 이름(값) 자체가 중요할 때 사용합니다. 가독성이 매우 좋습니다.
for ... in (객체 전용)
손님의 정보(속성:값)를 하나씩 꺼내볼 때 사용합니다. 배열에는 성능상 권장하지 않습니다.

  2. 마지막 방부터 1번 방까지 역순으로 점검하기
  3. 비밀번호 매칭 시스템 (do...while)
  4. VIP 손님(특정 조건) 발견 시 반복 중단하기 (break)
