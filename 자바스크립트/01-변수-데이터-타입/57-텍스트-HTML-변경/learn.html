DOM 콘텐츠 제어: textContent vs innerHTML

요소의 단순한 텍스트를 바꿀 때와 HTML 구조를 동적으로 생성할 때의 차이를 배웁니다.

속성 비교 요약

콘텐츠 변경 속성 비교
속성 용도 HTML 해석
textContent 텍스트만 변경 ❌ (태그도 글자로 출력)
innerHTML HTML 구조 포함 변경 ✅ (태그를 해석하여 렌더링)

학습 포인트

  • 보안성: 단순 텍스트라면 textContent를 쓰는 것이 성능과 보안(XSS 방지) 측면에서 더 안전합니다.
  • 템플릿 리터럴: innerHTML을 쓸 때는 백틱(`)을 활용해 가독성 있는 HTML 구조를 만드세요.
