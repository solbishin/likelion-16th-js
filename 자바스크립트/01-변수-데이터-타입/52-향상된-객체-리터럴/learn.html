더 스마트하게! 향상된 객체 리터럴

객체를 정의할 때 중복을 줄이고, 동적인 속성 이름을 사용하여 더 유연한 객체 구조를 만드는 기법을 배웁니다.

객체 리터럴의 3가지 변화

  • 속성 단축: 변수명과 속성명이 같으면 한 번만 적으세요.
  • 메서드 단축: 더 이상 : function을 고집할 필요가 없습니다.
  • 계산된 속성: 객체 생성 시점에 속성 키(Key)를 동적으로 결정하세요.

주요 기능 요약

객체 리터럴 개선 기능
기능 설명 예시
속성 할당 단축 속성명과 변수명이 같을 때 생략 { name }
메서드 단축 메서드 선언 시 function 생략 sayHi() { ... }
계산된 속성 이름 대괄호를 이용해 동적 키 생성 [key]: value

요약

향상된 객체 리터럴은 단순히 타이핑을 줄여주는 것 이상의 의미가 있습니다. 코드의 의도를 더 명확히 드러내고, 데이터 구조를 동적으로 다룰 수 있게 하여 더 강력한 JavaScript 프로그래밍을 가능하게 합니다.
